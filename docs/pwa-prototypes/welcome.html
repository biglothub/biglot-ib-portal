Implementation notes

  • Route: /welcome — server redirect ผู้ใช้ที่ยังไม่ login + เปิดบน mobile (matchMedia('(max-width: 767px)')) ครั้งแรกเท่านั้น เก็บ flag ใน localStorage key pwa.welcome.seen
  • Desktop isolation: หน้านี้ใช้ md:hidden wrapper ทั้งหมด — ถ้า desktop เผลอเปิด /welcome ให้ redirect ไป /auth/login เดิม
  • Frame 3 sign-in: reuse handler จาก src/routes/auth/login/+page.svelte — handleGoogleLogin() + email form ในหน้าใหม่
  • Hook เข้า PWA install flow: ถ้าเข้ามาจาก ?source=pwa ข้าม Frame 1-2 ไป Frame 3 ตรง ๆ (เป็น returning user ที่เพิ่งติดตั้ง)
  • Reuse pattern: status bar + safe-area + pill CTA + 3-dot pager → ใช้ใน IOSInstallSteps.svelte (TASK-102) ได้ด้วย
  • ขั้นต่อไป: ถ้า OK กับ direction นี้ → port เป็น Svelte component src/routes/welcome/+page.svelte + state machine ที่ใช้ $state() สำหรับ current frame index
