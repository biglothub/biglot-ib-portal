## CRITICAL Issues (Fix Immediately)

### C1. Prompt Injection in AI Chat
**File**: `src/routes/api/portfolio/ai-chat/+server.ts:66`
**Issue**: User-supplied message `role` field is cast as `'user' | 'assistant'` but NOT validated at runtime. A client can send `role: 'system'` to inject system prompts.
**Fix**: Add `if (msg.role !== 'user' && msg.role !== 'assistant') return 400`

### C2. IDOR in Journal API
**File**: `src/routes/api/portfolio/journal/+server.ts:11`
**Issue**: `client_account_id` comes from the request body without server-side verification. A malicious client could write journal entries to ANY account.
**Fix**: Use `getApprovedPortfolioAccount()` to get server-verified `account.id` instead.

### C3. PostgREST Filter Injection in Notebook Search
**File**: `src/routes/api/portfolio/notebook/+server.ts:185`
**Issue**: `.or(`title.ilike.%${query}%,content_plain.ilike.%${query}%`)` — user input is interpolated directly into a PostgREST filter string. Special characters like `,`, `.`, `(` can inject additional conditions.
**Fix**: Escape special PostgREST characters or use parameterized queries.

### C4. `trade_insights` Table Has Wide-Open RLS Policy
**File**: `supabase/migrations/014_trade_insights.sql:34`
**Issue**: `USING (true) WITH CHECK (true)` — ANY authenticated user can INSERT/UPDATE/DELETE any row.
**Fix**: Drop this policy. Service role already bypasses RLS.

### C5. `handle_new_user()` Allows Role Injection from Metadata
**File**: `supabase/migrations/009_save_google_avatar.sql`
**Issue**: `WHEN NEW.raw_user_meta_data->>'role' IN ('admin', 'master_ib', 'client')` — users can self-assign admin role via signup metadata.
**Fix**: Always default to `'client'`. Remove metadata role check.

### C6. `client_accounts` RLS Exposes `mt5_investor_password` Column
**File**: `supabase/migrations/001_full_schema.sql:264`
**Issue**: Client SELECT policy doesn't exclude `mt5_investor_password`. Even though passwords are encrypted, the encrypted blob is readable via direct Supabase queries.
**Fix**: Use a view or column-level security to exclude password from client queries.

## HIGH Issues

### H1. No Decrypt Input Length Validation
**File**: `src/lib/server/crypto.ts:34`
**Issue**: `Buffer.from(encoded, 'base64')` with no minimum length check. Short/empty input produces cryptic Node.js errors.
**Fix**: Add `if (combined.length < IV_LENGTH + TAG_LENGTH + 1) throw new Error('Invalid encrypted data')`

### H2. `is_own_account()` RLS Helper Doesn't Check Status
**File**: `supabase/migrations/001_full_schema.sql:231`
**Issue**: Suspended clients can still read all their trades, daily stats, positions, and equity snapshots.
**Fix**: Add `AND status = 'approved'` if business rules require it.

### H3. Notebook Error Handling — 5 Functions Missing `res.ok` Check
**File**: `src/routes/portfolio/notebook/+page.svelte:89-151`
**Issue**: `saveNote()`, `deleteNote()`, `restoreNote()`, `createFolder()`, `deleteFolder()` don't check response status or have try/catch. Users get no error feedback.

### H4. Admin Dashboard Unbounded Query
**File**: `src/routes/admin/+page.server.ts:18`
**Issue**: Fetches ALL `daily_stats` rows without limit, then deduplicates in JS. Will not scale.
**Fix**: Use a subquery, DISTINCT ON, or add `.limit()`.

### H5. XSS Risk in Analysis Page
**File**: `src/routes/portfolio/analysis/+page.svelte:349`
**Issue**: `{@html renderMarkdown(section.content)}` renders AI-generated content as HTML. While `escapeHtml()` is applied, the regex-based parser could miss edge cases.
**Fix**: Add DOMPurify sanitization on the final HTML output.

### H6. Keyboard Accessibility on Clickable Table Rows
**File**: `src/routes/portfolio/trades/+page.svelte:142`
**Issue**: `<tr onclick>` not focusable — keyboard users cannot navigate trade rows.
**Fix**: Add `tabindex="0"`, `role="button"`, `onkeydown` handler.

### H7. Memory Leaks in Notebook Page
**File**: `src/routes/portfolio/notebook/+page.svelte:53-59`
**Issue**: `saveTimer` and `searchTimer` debounce timers never cleaned up on component destroy.

### H8. `$derived()` Misuse (Should be `$derived.by()`)
**Files**: `TagManager.svelte:63`, `SidebarNews.svelte:34`
**Issue**: `$derived(() => { ... })` creates a function, not the result. Should be `$derived.by(() => { ... })`.

### H9. SidebarNews Store Subscription Memory Leak
**File**: `src/lib/components/layout/SidebarNews.svelte:30`
**Issue**: `marketNewsStore.subscribe()` never unsubscribes on destroy.

### H10. Analysis Streaming Has No Abort Signal
**File**: `src/routes/portfolio/analysis/+page.svelte:188-222`
**Issue**: If user navigates away during streaming, the reader continues in the background.

### H11. NotificationBell Missing Error Handling
**File**: `src/lib/components/layout/NotificationBell.svelte:13-61`
**Issue**: Initial fetch has no `.catch()`. `markAsRead`/`markAllAsRead` don't check for errors — local state gets out of sync.

### H12. Calendar Day Buttons Missing `aria-label`
**Files**: `day-view/+page.svelte:259`, `journal/+page.svelte:212`
**Issue**: Screen readers only hear the day number with no context.

### H13. Push Notification URL Not Validated
**Files**: `src/routes/api/push/send/+server.ts:42`, `src/service-worker.ts:108`
**Issue**: `url` from push payload passed directly to `openWindow()`. Could open phishing URLs.
**Fix**: Validate URL origin matches the app.

### H14. DayMiniPnlChart Has No Data Reactivity
**File**: `src/lib/components/charts/DayMiniPnlChart.svelte:26`
**Issue**: No `$effect` to update when `data` prop changes. Component shows stale data.

### H15. TradeContextChart Missing Null Check on `trade` Prop
**File**: `src/lib/components/portfolio/TradeContextChart.svelte:36`
**Issue**: `renderChart()` accesses `trade.open_time` etc. without checking if `trade` is defined.

### H16. IB Page Duplicate Data Loading
**Files**: `src/routes/ib/+page.server.ts`, `src/routes/ib/clients/+page.server.ts`
**Issue**: Nearly identical queries for clients + daily_stats with unbounded daily_stats query.

### H17. TagManager `deleteTag` Has No Error Handling
**File**: `src/lib/components/portfolio/TagManager.svelte:51`

### H18. `Math.max(...largeArray)` Stack Overflow Risk
**File**: `src/lib/server/stats-overview.ts:110-111`
**Issue**: `Math.max(...wins)` with 10,000+ trades will throw stack overflow.
**Fix**: Use `wins.reduce((max, v) => v > max ? v : max, -Infinity)`

## Positive Observations

1. **Auth pattern is solid**: `getUser()` JWT validation, proper cookie handling, profile-based role checks
2. **Svelte 5 migration complete**: Zero legacy `$:` reactive statements found. All components use Runes correctly.
3. **Trade ownership verification**: `verifyTradeOwnership()` is used consistently across all trade-related endpoints
4. **Chart cleanup**: All lightweight-charts instances properly call `chart.remove()` and `observer.disconnect()`
5. **RPC functions well-written**: `SECURITY DEFINER` with `auth.uid()` checks, `FOR UPDATE` locking, search path protection
6. **XSS protection in AI chat**: `AiChatMessage.svelte` properly escapes HTML before markdown processing
7. **MT5 encryption**: AES-256-GCM with random IV, proper auth tag handling
8. **Thai timezone handling**: `getBangkokToday()` and `toThaiDateString()` are consistent and correct (Thailand has no DST)
9. **Service worker**: Proper cache versioning, correct route bypasses, font caching
