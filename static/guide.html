คู่มือการใช้งาน IB Portal

คู่มือแนะนำการใช้งานระบบ IB Portal อย่างละเอียด สำหรับทุกบทบาท ตั้งแต่ Admin, Master IB ไปจนถึง Client

🛡

Admin

ผู้ดูแลระบบ — จัดการ Master IB, อนุมัติลูกค้า, ดูภาพรวมทั้งระบบ

  • Dashboard ภาพรวม KPI
  • อนุมัติ / ปฏิเสธลูกค้า
  • สร้างและจัดการ Master IB
  • ดูรายละเอียดพอร์ตลูกค้า
🤝

Master IB

พาร์ทเนอร์ IB — เพิ่มลูกค้า, ติดตามพอร์ต, ส่งข้อมูลเข้าสู่ระบบ

  • Dashboard สรุปลูกค้า
  • เพิ่มลูกค้าใหม่ (2 ขั้นตอน)
  • ดูสถานะและพอร์ตลูกค้า
  • ส่งข้อมูลลูกค้าใหม่หลังถูกปฏิเสธ
📈

Client

ลูกค้า/เทรดเดอร์ — ดูพอร์ต, ติดตามผลการเทรดแบบ Real-time

  • ดูยอดเงิน, Equity, P/L
  • กราฟ Equity Curve 30 วัน
  • ตาราง Open Positions
  • ประวัติการเทรดย้อนหลัง
🛡 คู่มือสำหรับ Admin

ผู้ดูแลระบบมีสิทธิ์เข้าถึงทุกส่วน รวมถึงการจัดการ Master IB และอนุมัติลูกค้า

ภาพรวมการทำงานร่วมกันระหว่าง Admin, Master IB, และ Client

Lifecycle ของ Client Account
1
Master IB ส่งข้อมูลลูกค้า
IB กรอกข้อมูลชื่อ, อีเมล, MT5 Account ID, Investor Password, Server แล้วส่งเข้าระบบ
สถานะ: pending
2
Admin ตรวจสอบและอนุมัติ
Admin เห็นรายชื่อใหม่ในหน้า Approvals ตรวจสอบข้อมูล MT5 Validation แล้วกดอนุมัติหรือปฏิเสธ
✓ อนุมัติ
สถานะ → approved
แจ้งเตือน IB + Client เข้าสู่ระบบได้
✗ ปฏิเสธ
สถานะ → rejected
แจ้งเตือน IB + แสดงเหตุผล
↺
กรณีถูกปฏิเสธ — IB ส่งข้อมูลใหม่
IB เห็นเหตุผลปฏิเสธ แก้ไข MT5 credentials แล้วกด "ส่งข้อมูลใหม่" สถานะจะกลับเป็น pending
3
Client เข้าสู่ระบบ
Client ใช้ Google OAuth เข้าสู่ระบบด้วยอีเมลที่ IB ลงทะเบียนไว้ ระบบจะ link กับ Client Account ที่ approved อัตโนมัติ
4
ทุกคนเห็นข้อมูลเทรด
Bridge Service ซิงค์ข้อมูล MT5 อัตโนมัติ ข้อมูลจะแสดงให้ทุก role ตามสิทธิ์ที่กำหนด
Admin: เห็นทุกลูกค้า
IB: เห็นลูกค้าตัวเอง
Client: เห็นพอร์ตตัวเอง
